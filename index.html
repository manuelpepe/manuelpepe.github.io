Traducción de aplicaciones

Posteado el 27-07-2015   |   En Fundamentos de la Programación   |  

Es el año 2015, plena era tecnologica. Con la velocidad con la que podemos compartir información por
todo el mundo es normal que necesitemos, en muchas ocaciones, traducir esa infromación a varios idiomas.

No todo el mundo tiene la suerte de conocer mas de un idioma, y es nuestro trabajo, como desarrolladores,
preocuparnos de que todo el mundo pueda utilizar nuestras aplicaciones de la forma mas cómoda y placentera
posible.

Hay varias maneras de resolver este problema, pero si estamos trabajando en una aplicación pequeña
y no queremos depender de una infraestructura externa a la nuestra (como Google API) o de una libreria
con cientos de lenguajes inecesarios, podemos construir nuestro propio y sencillo “traductor”.

Aviso: Esta definitivamente no es la mejor practica cuando nuestra aplicación tiene mucho texto, pero
si tiene poco texto, es una opción rapida y muy valida.

Aviso 2: Para estos ejemplos voy a usar Python, ya que me parece que es un lenguaje facil de entender,
incluso para los que nunca lo usaron.

Supongamos que tenemos una aplicación como la siguiente:

Ahora, ¿cómo hacemos para convertir nuestra aplicación en bilingüe y que nos den un mejor puntaje?
Al ser una aplicación con muy pocos textos yo haría lo siguiente:

Lo primero que hice fue crear dos arrays, uno con todos los textos del programa en español y otro con los mismos textos pero traducidos al inglés. Después de eso creé una variable y le asigné el idioma en el cual se va a ejecutar (én producción probablemente dejariamos al usuario elegir el idioma, pero para mantener el ejemplo lo más corto y limpio posible, lo puse yo).

Una vez elegido el idioma lo que hice fue lo siguiente:

  1. Borré el array que no se va a usar para no ocupar memoria inútilmente.
  2. Copié el contenido del array del idioma que se va a usar a una variable genérica DICT (la cual después va a usar el programa).
  3. Borré el contenido del array que usamos para no tenerlo dos veces en memoria ocupando recursos.

Luego de esto solo tuve que llamar, mediante un indice del array, el texto que necesitaba.

Cuales son las ventajas y desventajas de este metodo?

Ventajas:

  • Rápido: Tener el diccionario cargado en memoria constantemente hace que la respuesta sea muy rápida.
  • Simple de escribir: Sin contar las traducciones en sí, hay que agregar muy poco código al producto original para que funcione.
  • Ocupa poco espacio en el disco duro: Al no tener que agregar mucho código a nuestro proyecto, minimizamos el espacio que este ocupa en el disco duro.

Desventajas:

  • Consume mucha memoria: Si bien tener el diccionario cargado en memoria todo el tiempo puede hacer que las aplicaciones con poco texto respondan más rápido, si la aplicación tiene mucho texto, o la máquina poca memoria, podríamos ralentizar la máquina o, incluso, colgarla. De una u otra manera, estaríamos desperdiciando mucha memoria.
